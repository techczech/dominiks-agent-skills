# single-html-document

## Scope

Build polished single-file HTML documents from structured Markdown/YAML: reports, explainers, briefings, dashboards, browser slide decks, learning packs, review workbooks, feedback collectors, annotation tools, editable drafts, and component handbooks. Use when notes, research, meeting material, slide content, editorial drafts, QA plans, or component patterns need to become a self-contained document someone can email, share, open from disk, annotate, edit, review, or use offline. Use when the output should replace a PDF, PowerPoint, hosted dashboard, microsite, or form; inline assets; preserve interactivity; embed attachments; collect highlighted notes; export feedback JSON; or export reader-edit diffs. Trigger on single HTML file, shareable HTML document, portable explainer, browser-openable deck, feedback workbook, highlight and take notes, editable HTML draft, export a diff, self-contained website, or no build step.
